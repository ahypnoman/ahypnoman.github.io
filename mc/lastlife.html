Info

A configurable addon for Minecraft Bedrock that emulates the mod from Last Life

Docs

How to play:
  1. A world operator runs the @session command
  2. After a minute all players are randomly assigned an amount of lives* (default range is 2-6)
  3. After five minutes a random player is selected as the "Boogeyman". Their objective is to kill another player to be "cured"
*Lives are only assigned in the first session. There is a setting to grant a life to a random player every session (see "randomlifemode").
Objectives:
Green and yellow objective:
Survive.

Red objective:
Survive. Can also kill other players.

Bogeyman objective:
Kill another player
General commands:
These commands are prefixed by "@", not "/". Commands are case-sensitive, including usernames.

@help - Provides a shortened version of this text in chat.
@lives - Provides a list of players and how many lives they have.
@givelife <username> - Transfers a life from the caller to a specified player.
@sessionlogs <session number?> - The logs for a specified session. If item is not provided it return a list of available logs.
@info - Explains the Boogeyman's objective. Can only be run by the current Boogeyman.
Admin commands:
These commands are prefixed by "@", not "/". Commands are case-sensitive, including usernames. Only world operators can use these commands.

@boogeyman <username?> - Starts the game. Specifying a username will set that player as a Boogeyman*.
@options <help | ...> - Allows game rules to be set. Use "@options help" or see "@options commands" for a list of options.
@cure <code? | no reward?> - Cures the Boogeyman. Optional validation code. Optional "no reward" toggle (use an asterisk if true).
@session - starts a new session.
*Command to be used as a fallback if existing boogeyman leaves the game and for testing
@options commands:
These commands are prefixed by "@options ", not "/". Commands are case-sensitive, including usernames. Only world operators can use these commands.

@options help - Provides list of options commands.
@options player <username> lives get - Provides the lives of the specified player.
@options player <username> lives set <quantity?> - Sets the life count of the specified player.
@options bmmode get - Provides the current Boogeyman objective.
@options bmmode set <reward | punish | combined> - sets the current Boogeyman objective.
@options revivemode get - Provides the status of the revivemode gamerule.
@options revivemode set <true | false> - Provides the status of the revivemode gamerule.
@options randomlifemode get - Provides the status of the randomlifemode gamerule.
@options randomlifemode set <true | false> - Provides the status of the randomlifemode gamerule.
@options session get - Provides the current session number.
@options session set <integer> - Sets the current session number*.
@options liferange get - Gets the random life selection range.
@options liferange set <min | max> - Sets the random life range*.
*Setting to 0 and using the @session command will restart the game (it will not clear logs)
Admin instructions:
  • The game is divided into "sessions" that can be started by an admin using the "@session command". See "How to play" for information on sessions.
  • Some rules (such as the Boogeyman not being able to kill red names) must be enforced by an admin, like in the Last Life series.
  • The "bmmode" option determines the Boogeyman's reward/punishment for killing another player. It can be toggled between 3 modes: "reward" grants the Boogeyman a life when they kill another player, "punish" (default) sets the Boogeyman to their last life if they fail to kill another player, and "combined" is a combination of both other modes.
  • The "randomlifemode" option determines whether a random player gains a life when the next session starts. If is set to "true" a random player will gain a life every session. If it is set to "false" (default) lives cannot be created without admin intervention.
  • The "revivemode" option determines whether players can revive each other using the "@givelife" command. "false" is default.
  • The "liferange" option determines the range within which lives are randomly chosen at the start of the game and for any players that join later,
  • Sessions must be started manually.
  • Boogeyman trap kills are not recorded automatically. If someone claims to have trapped somebody as Boogeyman you can ask them for a validation code then run the command "@cure <code>" to cure them/validate that they are a Boogeyman (This does not validate that they actually trapped them, however).
  • If an admin wishes to cure the Boogeyman manually without a code the command "@cure *" is recommended as it will not reward the Boogeyman in "reward" or "combined" mode but will remove their status as Boogeyman.
  • Referring players to this site is recommended to explain rules and commands.
  • Session logs seen in the "@sessionlogs" command cannot be cleared and can be read by any player to prevent admin abuse. Don't cheat.
